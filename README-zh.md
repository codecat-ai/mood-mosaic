# Mood Mosaic

[English](README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md)

Mood Mosaic 是一个本地优先的情绪、精力和专注度日志，适合学生、创作者、远程工作者和引导师在不注册账号、不使用云同步的情况下进行私人反思。

## 问题

许多情绪应用依赖账号、云存储，或带有诊断导向的表达。习惯追踪器常常过于数字化，而电子表格又不适合快速记录每日背景。Mood Mosaic 将数据保存在浏览器中，并把简短的每日记录变成易读的日历马赛克和摘要。

## 功能

- 为选定日期添加或更新一条记录，包含情绪、精力、专注度和短备注。
- 查看按日期排序的彩色马赛克、平均值、情绪计数和连续记录天数，可按全部时间、选定日期所在的周一至周日，或选定日历月筛选。
- 在紧凑且便于键盘操作的近期记录列表中回顾记录，并直接进入编辑。
- 在记录表单中使用日期快捷键：`T` 回到今天，`[` 跳到上一个已保存日期，`]` 跳到下一个已保存日期。
- 复制简洁的反思摘要，用于日志、教练沟通或例行检查。
- 导出 JSON 备份，并在替换浏览器数据前预览导入内容。
- 校验记录和导入内容，遇到无效或旧版数据时不会崩溃。
- 生产构建后完全在客户端运行。

## 从源码安装

Mood Mosaic 未发布到包注册表。请使用源码检出：

```bash
git clone https://github.com/codecat-ai/mood-mosaic.git
cd mood-mosaic
npm ci
```

## 快速开始

```bash
npm run dev
```

打开 Vite 输出的本地地址。选择今天的日期或较早的记录日期，填写情绪、精力、专注度和备注，然后通过马赛克和摘要面板查看模式。
Trend range 控件会按全部时间、包含所选记录日期的周一至周日，或所选日历月筛选快照、马赛克、近期记录和复制摘要。也可以使用近期记录列表，把某一天直接载入表单，而不必手动使用日期选择器。未在输入框中打字时，键盘用户可以按 `T` 回到今天，按 `[` 移动到上一个已保存日期，按 `]` 移动到下一个已保存日期。

## 示例

记录示例：

```json
{
  "date": "2026-05-21",
  "mood": "focused",
  "energy": 4,
  "focus": 5,
  "note": "Finished the project draft",
  "schemaVersion": 1
}
```

备份示例：

```json
{
  "schemaVersion": 1,
  "entries": []
}
```

回顾流程示例：保存多天记录后，在 Trend range 中选择 This week 或 This month，用 Tab 移动到近期记录列表，再通过 Edit 按钮或 `[`、`]` 快捷键把某个日期的情绪、精力、专注度和备注载入表单。

## 配置

项目没有账号、服务器或云端配置。数据存储在浏览器 LocalStorage 的 `mood-mosaic:journal` 键下。如果需要便携备份或迁移浏览器，请定期导出 JSON。导入时会先显示预览，包含可接受记录数、日期范围、问题列表，以及会替换和新增的日期数量；确认后才会替换浏览器数据。

## 开发

```bash
npm run lint
npm run typecheck
npm test -- --run
npm run build
```

核心模型在 `src/journal.ts`，近期记录选择在 `src/recentEntries.ts`，导入预览逻辑在 `src/importPreview.ts`，分析和趋势筛选逻辑在 `src/summary.ts`，存储适配器在 `src/storage.ts`，React UI 在 `src/App.tsx`。
