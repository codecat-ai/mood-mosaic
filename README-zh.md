# Mood Mosaic

[English](README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md)

Mood Mosaic 是一个本地优先的情绪、精力和专注度日志，适合学生、创作者、远程工作者和引导师在不注册账号、不使用云同步的情况下进行私人反思。

## 问题

许多情绪应用依赖账号、云存储，或带有诊断导向的表达。习惯追踪器常常过于数字化，而电子表格又不适合快速记录每日背景。Mood Mosaic 将数据保存在浏览器中，并把简短的每日记录变成易读的日历马赛克和摘要。

## 功能

- 为选定日期添加或更新一条记录，包含情绪、精力、专注度和短备注。
- 查看按日期排序的彩色马赛克、平均值、情绪计数和连续记录天数，可按全部时间、选定日期所在的周一至周日，或选定日历月筛选。
- 在紧凑且便于键盘操作的近期记录列表中按日期、情绪或备注搜索，并直接进入编辑。
- 在记录表单中使用日期快捷键：`T` 回到今天，`[` 跳到上一个已保存日期，`]` 跳到下一个已保存日期。
- 将未保存的表单编辑重置为所选日期的已保存记录；如果该日期没有记录，则回到默认空状态。
- 当表单不同于所选日期的已保存记录或默认状态时，显示轻量的未保存更改提示。
- 在保存前看到轻量表单提示，提醒无效日期、缺少评分，以及备注接近已保存记录长度上限。
- 在备注字段旁选择可选反思提示，并可把提示追加为备注开头，而不会替换已输入文字。
- 复制简洁的反思摘要，用于日志、教练沟通或例行检查。
- 导出 JSON 备份，并提供首次备份提示、导入试运行示例；在替换浏览器数据前预览导入内容。
- 在生成备份或刷新恢复预览后，看到紧凑的生成/刷新时间。
- 校验记录和导入内容，遇到无效或旧版数据时不会崩溃。
- 生产构建后完全在客户端运行。

## 从源码安装

Mood Mosaic 未发布到包注册表。请使用源码检出：

```bash
git clone https://github.com/codecat-ai/mood-mosaic.git
cd mood-mosaic
npm ci
```

## 快速开始

```bash
npm run dev
```

打开 Vite 输出的本地地址。选择今天的日期或较早的记录日期，填写情绪、精力、专注度和备注，然后通过马赛克和摘要面板查看模式。
如果所选日期无效、评分不在 1 到 5 之间，或备注接近已保存记录长度上限，表单旁会显示轻量提示。当当前表单不同于所选日期的已保存记录或默认值时，会显示 Unsaved changes 提示，并在 Save 或 Reset form 后消失。使用 Reset form 可以放弃所选日期的未保存编辑；它会重新载入该日期的已保存值，若没有记录则回到默认值，并且不会改动已存记录。可选的 Reflection prompt 选择器能显示简短的情绪、精力、专注度或下一步问题，其按钮会把提示追加到备注中，而不是覆盖已输入文字。Trend range 控件会按全部时间、包含所选记录日期的周一至周日，或所选日历月筛选快照、马赛克、近期记录和复制摘要。Search recent entries 只会在当前趋势范围内按日期、情绪或备注缩小近期记录回顾列表；也可以从筛选后的列表把某一天直接载入表单，而不必手动使用日期选择器。未在输入框中打字时，键盘用户可以按 `T` 回到今天，按 `[` 移动到上一个已保存日期，按 `]` 移动到下一个已保存日期。

## 示例

记录示例：

```json
{
  "date": "2026-05-21",
  "mood": "focused",
  "energy": 4,
  "focus": 5,
  "note": "Finished the project draft",
  "schemaVersion": 1
}
```

备份示例：

```json
{
  "schemaVersion": 1,
  "entries": []
}
```

导入试运行示例：

```json
{
  "schemaVersion": 1,
  "entries": [
    {
      "date": "2026-05-20",
      "mood": "steady",
      "energy": 4,
      "focus": 4,
      "note": "Dry-run replacement"
    },
    {
      "date": "2026-05-21",
      "mood": "bright",
      "energy": 5,
      "focus": 5,
      "note": "Dry-run new date"
    }
  ]
}
```

如果 `2026-05-20` 已经有记录，试运行预览会报告接受 2 条记录，日期范围为
`2026-05-20` 到 `2026-05-21`，其中 1 个已有日期会被替换，1 个新日期会被新增。

回顾流程示例：保存多天记录后，在 Trend range 中选择 This week 或 This month，用 Tab 移动到 Search recent entries，输入日期、情绪或备注的一部分，再通过 Edit 按钮或 `[`、`]` 快捷键把某个日期的情绪、精力、专注度和备注载入表单。

备份/恢复检查示例：点击 Export JSON，确认备份面板显示该 JSON 的生成时间；再预览一次导入，确认恢复预览在任何已存记录改变前显示刷新时间。

## 配置

项目没有账号、服务器或云端配置。数据存储在浏览器 LocalStorage 的 `mood-mosaic:journal` 键下。如果需要便携备份或迁移浏览器，请定期导出 JSON。没有已保存记录时，备份面板会提示先保存记录，导出的 JSON 才有实际意义。导出后，备份面板会为当前可见 JSON 显示紧凑的生成时间。导入区域会显示简洁的恢复安全提示，并先显示预览，包含可接受记录数、日期范围、问题列表、会替换和新增的日期数量，以及紧凑的刷新时间；确认后才会替换浏览器数据。导入面板还包含一个小型只读试运行 JSON 示例，方便首次使用者在粘贴自己的备份前理解替换和新增日期数量。

npm run build
