# Mood Mosaic

[English](README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md)

Mood Mosaic は、学生、作り手、リモートワーカー、ファシリテーター向けのローカルファーストな気分、エネルギー、集中度のジャーナルです。アカウントやクラウド同期なしで個人的な振り返りを残せます。

## 課題

多くの気分記録アプリはアカウントやクラウド保存に依存していたり、診断を連想させる表現を含んでいたりします。習慣トラッカーは数値中心になりがちで、スプレッドシートは短い日々の文脈を書くには扱いにくいです。Mood Mosaic はデータをブラウザーに保存し、小さな日次記録を読みやすいカレンダーモザイクと短い要約に変えます。

## 機能

- 選択した日付に気分、エネルギー、集中度、短いメモを追加または更新。
- 日付順のカラーモザイク、平均、気分の件数、連続記録日数を、全期間、選択日を含む月曜から日曜の週、または選択した暦月で絞り込んで表示。
- キーボードで扱いやすいコンパクトな最近の記録リストを日付、気分、メモで検索し、そのまま編集へ移動。
- 記録フォームで日付ショートカットを使用: `T` で今日、`[` で前の保存済み日付、`]` で次の保存済み日付へ移動。
- 未保存のフォーム編集を、選択日の保存済み記録へリセット。記録がない日は既定の空状態へ戻します。
- フォームが選択日の保存済み記録または既定値と異なる間、軽い未保存変更インジケーターを表示。
- 保存前に、無効な日付、未入力または範囲外の評価、保存できる長さの上限に近いメモを軽いフォームヒントで確認。
- メモ欄の近くで任意の振り返りプロンプトを選び、既存のメモを置き換えずに書き出しとして追加。
- ジャーナル、コーチ、チェックイン用に簡潔な振り返り要約をコピー。
- 初回向けの案内とインポートのドライラン例付きで JSON バックアップをエクスポートし、ブラウザーデータを置き換える前に貼り付けた JSON の sanity check とインポート内容を確認。
- バックアップ生成後や復元プレビュー更新後に、コンパクトな生成時刻と更新時刻を確認。
- 復元を確定する前に、プレビューが既存日付を置き換えるのか、新しい日付を追加するのか、またはその両方なのかを示すコンパクトな判断メモを確認。
- 無効なデータや旧 schema のデータを扱ってもクラッシュしない検証。
- 本番ビルド後は完全にクライアント側で動作。

## ソースからのインストール

Mood Mosaic はパッケージレジストリでは公開していません。ソースチェックアウトを使ってください。

```bash
git clone https://github.com/codecat-ai/mood-mosaic.git
cd mood-mosaic
npm ci
```

## クイックスタート

```bash
npm run dev
```

Vite が表示するローカル URL を開きます。今日の日付または過去の記録日を選び、気分、エネルギー、集中度、メモを入力し、モザイクと要約パネルで傾向を確認します。
選択日が無効、評価が 1 から 5 の範囲外、またはメモが保存できる長さの上限に近い場合は、フォームの近くに軽いヒントが表示されます。現在のフォームが選択日の保存済み記録または既定値と異なる場合は Unsaved changes インジケーターが表示され、Save または Reset form の後に消えます。Reset form を使うと、選択日の未保存編集を破棄し、その日の保存済み値を再読み込みできます。記録がない場合は既定値へ戻り、保存済み記録は変更されません。任意の Reflection prompt セレクターでは、気分、エネルギー、集中度、次の一歩に関する短い質問を表示でき、ボタンで既存の入力を上書きせずにメモへ追加できます。Trend range コントロールは、スナップショット、モザイク、最近の記録、コピーされる要約を、全期間、選択した記録日を含む月曜から日曜の週、または選択した暦月で絞り込みます。Search recent entries は、現在の傾向範囲内で最近の記録リストだけを日付、気分、メモで絞り込みます。絞り込んだリストから、日付ピッカーを手動で操作せずにその日の記録をフォームへ読み込めます。入力欄で文字を入力していないときは、`T` で今日、`[` で前の保存済み日付、`]` で次の保存済み日付へ移動できます。バックアップ JSON をインポート欄に貼り付けると、Preview import をクリックする前に読み取り専用の Backup sanity check が JSON の有効性、schema バージョン、元の記録数、有効な記録数、問題、日付範囲を要約します。プレビュー後は、Confirm import の近くにある読み取り専用の復元判断メモが、確定すると既存日付を置き換えるのか、新しい日付を追加するのか、またはその両方なのかを示します。

## 例

記録例:

```json
{
  "date": "2026-05-21",
  "mood": "focused",
  "energy": 4,
  "focus": 5,
  "note": "Finished the project draft",
  "schemaVersion": 1
}
```

バックアップ例:

```json
{
  "schemaVersion": 1,
  "entries": []
}
```

インポートのドライラン例:

```json
{
  "schemaVersion": 1,
  "entries": [
    {
      "date": "2026-05-20",
      "mood": "steady",
      "energy": 4,
      "focus": 4,
      "note": "Dry-run replacement"
    },
    {
      "date": "2026-05-21",
      "mood": "bright",
      "energy": 5,
      "focus": 5,
      "note": "Dry-run new date"
    }
  ]
}
```

`2026-05-20` の記録がすでにある場合、このドライランのプレビューは
`2026-05-20` から `2026-05-21` までの 2 件を受け入れ、既存日付 1 件を
置き換え、新しい日付 1 件を追加すると表示します。

レビュー手順の例: 複数日の記録を保存し、Trend range で This week または This month を選び、Tab で Search recent entries へ移動して日付、気分、メモの一部を入力し、Edit ボタンまたは `[`、`]` ショートカットからその日の気分、エネルギー、集中度、メモをフォームへ読み込みます。

バックアップ/復元確認の例: Export JSON をクリックして、バックアップパネルにその JSON の生成時刻が表示されることを確認します。インポート JSON を貼り付けて Backup sanity check を確認し、次にインポートをプレビューして、保存済み記録が変わる前に復元プレビューの更新時刻と、確定後に既存日付を置き換えるのか、新しい日付を追加するのか、またはその両方なのかを確認します。

デモスクリーンショット fixture: `src/demoFixtures.ts` は、決定的なジャーナル記録と、ドキュメントや視覚チェック向けの純粋関数 `buildDemoScreenshotChecklist()` をエクスポートします。安定したタイトル、日付範囲、記録件数、推奨される UI 状態、エスケープ済みかつ秘匿情報を除いたメモプレビューを返すため、開発サーバーやブラウザー自動化なしでスクリーンショットと README の例をそろえられます。

## 設定

アカウント、サーバー、クラウド設定はありません。データはブラウザー LocalStorage の `mood-mosaic:journal` に保存されます。持ち運べるバックアップやブラウザー移行が必要な場合は、JSON を定期的にエクスポートしてください。保存済みの記録がない場合、バックアップパネルは意味のあるエクスポートには先に記録を保存する必要があることを案内します。エクスポート後は、表示中の JSON に対するコンパクトな生成時刻がバックアップパネルに表示されます。インポート欄には短い復元時の安全案内が表示され、受け入れ可能な記録数、日付範囲、問題一覧、置き換えられる日付数、新しく追加される日付数、コンパクトな復元判断メモ、更新時刻を先に表示し、確認後にブラウザーデータを置き換えます。復元判断メモは、確定後に既存日付を置き換えるのか、新しい日付を追加するのか、またはその両方なのかを明示します。インポートパネルには小さな読み取り専用のドライラン JSON 例もあり、初めて使う人が自分のバックアップを貼り付ける前に置き換え数と新規日付数を確認できます。自分の JSON を貼り付けると、読み取り専用の sanity check が有効性、schema バージョン、元の記録数、有効な記録数、問題、日付範囲を保存せずに要約し、完全なプレビュー手順はそのまま残ります。

## 開発

```bash
npm run lint
npm run typecheck
npm test -- --run
npm run build
```

コアモデルは `src/journal.ts`、記録ヒントのロジックは `src/entryValidationHints.ts`、任意のメモプロンプトロジックは `src/notePrompts.ts`、決定的なフォームリセットロジックは `src/formReset.ts`、最近の記録選択は `src/recentEntries.ts`、インポートプレビューは `src/importPreview.ts`、バックアップ sanity check は `src/backupSanityCheck.ts`、分析と傾向フィルターは `src/summary.ts`、ストレージアダプターは `src/storage.ts`、React UI は `src/App.tsx` にあります。デモスクリーンショット fixture は `src/demoFixtures.ts` にあります。

## テスト

テストは検証と upsert、記録ヒントの動作とアクセシビリティ連携、最近の記録の並び順、件数制限、検索、分析とモザイクの並び順、ストレージのインポートとエクスポート、インポートプレビュー、旧 schema の正規化、バックアップ sanity check、メモプロンプトの検索と追加、日付ごとの編集、傾向フィルター、未保存フォームのリセット、未保存変更の検出、UI のスモークフローを対象にしています。バックアップ時刻の整形は、決定的な純粋 helper テストと注入した時刻を使う UI テストでカバーしています。復元判断メモは純粋 helper テストと、確定前にプレビューがストレージを変更しないことを確認する UI テストでカバーしています。日付ショートカットは純粋な helper テストと UI のステータス更新テストでカバーしています。デモ fixture のテストは、決定的な記録、日付範囲、メモプレビューの安全性、切り詰め、クローン安全性をカバーしています。振る舞いを変える前にテストを書いてください。
